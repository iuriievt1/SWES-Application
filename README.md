SWES Reservation Module UI (Front-End Assessment)
1) Implementation of selected use cases for the SWES Reservation Module assessment:
- UC-R1: Create Reservation (form + client-side validation + REST submit)
- UC-R5: Equipment Overview & History (table/cards + filters + sorting + pagination + URL state)
- UC-R12: Calendar View (available/unavailable days + click to prefill)
- UC-R13: API Call Simulation email notification (simulate sending via api/notify + success/error UI)
- Focus: front-end logic, UI components, and user interactions. Backend is simulated via Next.js route handlers

2) Tech Stack:
- Next.js (App Router) + TypeScript
- Bootstrap 5 (utility + components)
- No external backend required (REST is simulated via api/routes)

3) Getting Started (Run Locally)
- Requirements:
- Node.js 18+
- npm (or pnpm/yarn)

4) Install & Run:
npm install
npm run dev
